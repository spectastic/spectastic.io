How each limiter decides
Token bucket

A bucket of limit tokens refills at limit ÷ window per second. Each request spends a token; an empty bucket means 429. Naturally allows bursts up to the bucket size — the choice the plan made.

Sliding window

Counts the requests in the trailing window seconds; once that count hits limit, further requests get 429 until the oldest ages out. Smoother, but no burst headroom.
